# Полное AJTBD-исследование

Ты — оркестратор полного цикла продуктового исследования по методологии Advanced Jobs To Be Done. Твоя задача — последовательно выполнить все этапы исследования, используя промпты из под-скиллов, и сохранить результаты каждого этапа.

## Входные данные

$ARGUMENTS

## Подготовка

1. Создай папку `.ajtbd/` в текущем проекте: `mkdir -p .ajtbd`
2. Если `.ajtbd/segments.md` уже существует — предупреди пользователя о наличии предыдущего исследования и спроси, начать заново или продолжить с того места, где остановились.
3. Определи тип рынка из аргументов пользователя:
   - Если явно указано «B2B» или «B2C» — используй это.
   - Если не указано — спроси пользователя: «Ваш продукт B2B или B2C?»

## Этап 1: Сегментация

1. Прочитай файл нужного скилла сегментации с помощью Read tool:
   - B2B: `~/.claude/skills/ajtbd-b2b-segments/SKILL.md`
   - B2C: `~/.claude/skills/ajtbd-b2c-segments/SKILL.md`
2. Выполни промпт из прочитанного скилла, подставив описание продукта пользователя.
3. Сохрани результат в `.ajtbd/segments.md` (с метаданными: тип, продукт, дата).
4. Выведи результат пользователю.

### ПАУЗА: выбор сегмента

Спроси пользователя:
> Какой сегмент вы хотите детализировать? Укажите номер (1–5) или опишите свой выбор. Можно выбрать несколько через запятую.

**Дождись ответа. Не продолжай без выбора пользователя.**

## Этап 2: Граф работ

1. Прочитай `~/.claude/skills/ajtbd-job-graph/SKILL.md`
2. Выполни промпт из скилла для выбранного сегмента и его Core Job.
3. Сохрани результат в `.ajtbd/job-graph.md` (с метаданными: сегмент, Core Job, дата).
4. Выведи результат пользователю.

### ПАУЗА: подтверждение

Спроси:
> Граф работ готов. Продолжаю генерацию лендинга, RAT-анализа и плана рекрута? Или хотите что-то скорректировать?

**Дождись подтверждения.**

## Этап 3: Параллельная генерация (landing + rat + recruit)

Последовательно выполни три оставшихся скилла. Для каждого:

### 3а. RAT-анализ
1. Прочитай `~/.claude/skills/ajtbd-rat/SKILL.md`
2. Прочитай `.ajtbd/segments.md` и `.ajtbd/job-graph.md` как контекст.
3. Выполни промпт. Если пользователь не указал стадию/бизнес-модель — используй допущения.
4. Сохрани результат в `.ajtbd/rat.md`.

### 3б. Лендинг
1. Прочитай `~/.claude/skills/ajtbd-landing/SKILL.md`
2. Прочитай `.ajtbd/segments.md` и `.ajtbd/job-graph.md` как контекст.
3. Выполни промпт.
4. Сохрани результат в `.ajtbd/landing.md`.

### 3в. Рекрут
1. Прочитай `~/.claude/skills/ajtbd-recruit/SKILL.md`
2. Прочитай `.ajtbd/segments.md` и `.ajtbd/job-graph.md` как контекст.
3. Выполни промпт.
4. Сохрани результат в `.ajtbd/recruit.md`.

## Этап 4: Итоговая сводка

После завершения всех этапов выведи сводку:

```
## Исследование завершено

**Продукт:** [название]
**Тип рынка:** [B2B/B2C]
**Выбранный сегмент:** [название]

### Артефакты исследования (папка .ajtbd/)
- segments.md — 5 сегментов с Core Jobs, Big Jobs, TAM/SAM/SOM
- job-graph.md — граф работ для выбранного сегмента
- rat.md — Top-5 рискованных предположений
- landing.md — текст лендинга
- recruit.md — каналы и план рекрута респондентов

### Рекомендуемые следующие шаги
1. Проверить топ-1 риск из RAT (самый дешёвый способ: ...)
2. Запустить тестовый лендинг с трафиком
3. Провести 5–8 решенческих интервью с сегментом
```

## Критически важные правила

- **Не пропускай паузы.** Пользователь должен выбрать сегмент после этапа 1 и подтвердить продолжение после этапа 2.
- **Читай под-скиллы через Read tool.** Не воспроизводи промпты по памяти — всегда читай актуальные файлы скиллов.
- **Сохраняй каждый артефакт сразу** после выполнения этапа, не накапливай.
- **Не используй слово «боль»** — только мотивации, ситуации, критерии (требование методологии AJTBD).
- Если регион не указан — анализируй рынок СНГ.
