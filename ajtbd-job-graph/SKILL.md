## Системные инструкции: работа с артефактами AJTBD

**Зависимости: требуется `.ajtbd/segments.md`**

### Перед началом работы
1. Проверь наличие файла `.ajtbd/segments.md` с помощью Read tool.
2. **Если файла нет** — ОСТАНОВИСЬ и выведи пользователю:
   > Для построения графа работ необходимо сначала провести сегментацию.
   > Запустите `/ajtbd-b2b-segments` или `/ajtbd-b2c-segments` с описанием вашего продукта.
   Не продолжай выполнение скилла.
3. **Если файл есть** — прочитай его и используй как контекст. Выведи пользователю список сегментов из файла и спроси, для какого сегмента и какой Core Job строить граф (если пользователь не указал это в аргументах).

### Входные данные
- Если пользователь передал `$ARGUMENTS` — используй их как указание на сегмент/Core Job.
- Если аргументы пустые — предложи выбрать из сегментов в `.ajtbd/segments.md`.

### После выполнения
Сохрани полный результат в файл `.ajtbd/job-graph.md` с помощью Write tool. В начало файла добавь метаданные:
```
# Граф работ AJTBD
- Сегмент: [название выбранного сегмента]
- Core Job: [описание Core Job]
- Дата: [текущая дата]
---
```
Затем полный текст результата.

Сообщи пользователю: «Результат сохранён в `.ajtbd/job-graph.md`. Теперь доступны следующие шаги:
- `/ajtbd-landing` — текст лендинга
- `/ajtbd-rat` — анализ рисков (RAT)
- `/ajtbd-recruit` — каналы рекрута респондентов»
